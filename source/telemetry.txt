
===================
Configure Telemetry
===================

.. default-domain:: mongodb

.. contents:: On this page
   :local:
   :backlinks: none
   :depth: 1
   :class: singlecol

{+atlas-cli+}'s telemetry collects anonymous, aggregated usage data to
help MongoDB focus on improvements with the greatest impact. The 
{+atlas-cli+} enables telemetry by default.

Learn What the {+atlas-cli+} Tracks
-----------------------------------

When telemetry is enabled, {+atlas-cli+} tracks non-Personally-Identifiable Information (PII), which includes but is not limited to:

.. list-table::
   :header-rows: 1
   :widths: 60 40

   * - Data
     - Example Value

   * - {+atlas-cli+} version number
     - ``1.0.0``
  
   * - Installation source
     - ``Homebrew``

   * - Operating System (OS) and OS version
     - ``Windows 11.5``

   * - Authentication method. The values for the |api| keys and login
       credentials are *not* tracked.
     - ``API key``   

   * - Select details for commands you run. Arguments are tracked only
       if they use pre-set allowable values, such as cluster regions.
     - ``timestamp: 2022-04-11T11:35:46.794119+01:00``
       ``command: atlas cluster create``
       ``--provider AWS --region US_EAST_1``

   * - Performance information, such as amount of time it takes for the
       {+atlas-cli+} to execute a command.
     - ``completion timestamp: 2022-04-11T11:35:49.456719+01:00``

   * - Errors you encounter, including the command you run and the
       parameters you use.
     - ``atlas rgister``
       ``Error: unknown command "rgister" for "atlas"``

Learn What the {+atlas-cli+} Doesn't Track
------------------------------------------

The {+atlas-cli+} *doesn't* track:

.. list-table::
   :header-rows: 1
   :widths: 60 40

   * - Data
     - Example

   * - PII and values that could potentially contain PII, including all
       free text-entry fields such as custom names or database user
       names.
     - ``--clusterName MyCluster``

   * - |api| key values or |service| login credentials.
     - ``private_api_key abcdefghi123456789``


Disable Telemetry for the {+atlas-cli+}
---------------------------------------

To disable telemetry for the {+atlas-cli+}, run the following command in the terminal:

.. code-block:: sh

   atlas config set telemetry_enabled false

You can also disable telemetry using the :ref:`telemetry environment variable <atlas-cli-env-vars>`. 


Enable Telemetry for the {+atlas-cli+}
--------------------------------------

The {+atlas-cli+} enables telemetry by default. If telemetry is currently disabled, you can enable telemetry by running the following command in the terminal:

.. code-block:: sh

   atlas config set telemetry_enabled true

You can also enable telemetry using the :ref:`telemetry environment variable <atlas-cli-env-vars>`.
